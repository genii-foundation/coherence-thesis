Technological capture arises when a core function depends on infrastructure governed by someone else. A vendor can change terms, revoke access, expose data, or make migration prohibitively expensive.

Providence should identify which functions require common governance and which can rely on replaceable services. Every external dependency needs an owner, exit plan, data export, security review, and account of the authority it gives the provider.
